# AutoBay 快速启动指南

## 前置条件

| 工具 | 最低版本 | 用途 |
|------|---------|------|
| Python | 3.12+ | 后端运行 |
| Node.js | 20+ | 前端运行 |
| Docker & Docker Compose | 最新 | PostgreSQL / Redis / MinIO |
| Git | 任意 | 版本管理 |

---

## 第一步：环境变量配置

```bash
cd auto-bay
cp .env.example .env
```

打开 `.env` 文件，**必须修改**以下配置：

```ini
# 安全密钥（随机生成，必改）
SECRET_KEY=用这个命令生成: python -c "import secrets; print(secrets.token_urlsafe(32))"
JWT_SECRET_KEY=用这个命令生成: python -c "import secrets; print(secrets.token_urlsafe(32))"

# AI 功能需要（至少配一个）
ANTHROPIC_API_KEY=sk-ant-xxx      # 从 console.anthropic.com 获取
OPENAI_API_KEY=sk-xxx             # 从 platform.openai.com 获取（可选）
```

其他配置保持默认即可用于本地开发。

---

## 第二步：启动基础设施服务

```bash
docker compose -f docker-compose.dev.yml up -d
```

这会启动：
- **PostgreSQL 16** — `localhost:5432`（用户名/密码: autobay/autobay）
- **Redis 7** — `localhost:6379`
- **MinIO** — `localhost:9000`（控制台: `localhost:9001`，用户名/密码: minioadmin/minioadmin）

验证服务状态：
```bash
docker compose -f docker-compose.dev.yml ps
```

---

## 第三步：安装后端依赖

```bash
cd backend

# 方式一：使用 uv（推荐，更快）
pip install uv
uv sync

# 方式二：使用 pip
pip install -e ".[dev]"
```

---

## 第四步：数据库迁移

```bash
cd backend

# 生成初始迁移文件
alembic revision --autogenerate -m "initial schema"

# 执行迁移（创建所有表）
alembic upgrade head
```

验证表是否创建成功：
```bash
# 连接到 PostgreSQL
docker exec -it auto-bay-postgres-1 psql -U autobay -d autobay -c "\dt"
```

应该看到 16 张表：
```
users, platform_connections, products, product_translations,
listings, orders, order_line_items, shipments,
inventory_locations, inventory_items, inventory_movements,
exchange_rates, price_history, customer_messages,
marketing_campaigns, ai_tasks
```

---

## 第五步：启动后端 API

```bash
cd backend

# 开发模式（自动重载）
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

验证：打开浏览器访问 http://localhost:8000/docs

你应该看到 Swagger UI，包含所有 API 端点。

---

## 第六步：启动 Celery Worker（后台任务）

打开**新的终端窗口**：

```bash
cd backend

# 启动 Worker（处理订单同步、库存同步等）
celery -A app.workers.celery_app worker -l info -Q default,sync,ai

# （可选）另开一个终端启动 Beat（定时调度）
celery -A app.workers.celery_app beat -l info
```

---

## 第七步：安装并启动前端

```bash
cd frontend

# 安装依赖
npm install

# 开发模式
npm run dev
```

打开浏览器访问 http://localhost:3000

---

## 第八步：创建第一个用户

```bash
# 通过 API 注册
curl -X POST http://localhost:8000/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email": "admin@autobay.com", "password": "your-password", "full_name": "Admin"}'

# 登录获取 token
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "admin@autobay.com", "password": "your-password"}'
```

或者直接在前端 http://localhost:3000/login 注册和登录。

## 完整服务架构（本地开发）

```
http://localhost:3000   ← Next.js 前端（Dashboard）
http://localhost:8000   ← FastAPI 后端（API）
http://localhost:8000/docs ← Swagger API 文档
http://localhost:5432   ← PostgreSQL 数据库
http://localhost:6379   ← Redis 缓存/队列
http://localhost:9000   ← MinIO 对象存储
http://localhost:9001   ← MinIO 管理控制台
```

## 使用 Makefile 快捷命令

```bash
make install          # 安装前后端全部依赖
make dev-backend      # 启动后端
make dev-frontend     # 启动前端
make dev-worker       # 启动 Celery Worker
make dev-beat         # 启动 Celery Beat
make migrate          # 执行数据库迁移
make migrate-new msg="description"  # 新建迁移
make test             # 运行测试
make lint             # 代码检查
make format           # 代码格式化
make docker-up        # Docker 全部启动（生产模式）
make docker-down      # Docker 全部停止
```

## 常见问题

### Q: 数据库连接失败
确认 Docker 中 PostgreSQL 已启动：
```bash
docker compose -f docker-compose.dev.yml logs postgres
```

### Q: Redis 连接失败
确认 Redis 已启动：
```bash
docker compose -f docker-compose.dev.yml logs redis
```

### Q: 前端 API 请求 401
检查 `.env` 中 `JWT_SECRET_KEY` 是否与后端启动时一致。前端默认连接 `http://localhost:8000/api/v1`。

### Q: AI 功能不工作
确认 `.env` 中 `ANTHROPIC_API_KEY` 已正确配置。

### Q: Celery 任务不执行
1. 确认 Redis 已启动
2. 确认 Worker 和 Beat 都在运行
3. 查看 Worker 终端的日志输出
