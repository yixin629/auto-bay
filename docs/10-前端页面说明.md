# AutoBay 前端页面说明

## 技术栈

- **框架**：Next.js 15 (App Router)
- **语言**：TypeScript
- **样式**：Tailwind CSS
- **组件库**：shadcn/ui (Button, Card, Badge, Table, Input, Separator)
- **图标**：Lucide React
- **API 客户端**：自定义 fetch wrapper (`lib/api.ts`)

## 页面详情

### Dashboard 首页
- 4 个 KPI 卡片：产品数、活跃 Listing、今日订单、月收入
- 平台连接状态卡片（eBay AU / Amazon US / Shopify / TikTok Shop）
- 最近订单预览

### Products 产品列表
- 数据表格：SKU、标题、分类、货源模式、成本、状态
- 状态徽章：draft(黄) / active(绿) / archived(灰)
- "Add Product" 按钮
- 从 API 加载数据，显示 loading 和空状态

### Listings 跨平台 Listing
- 数据表格：标题、平台、区域、价格、状态、外部 ID
- 状态徽章：draft/pending/active/paused/error/ended

### Orders 订单列表
- 数据表格：订单号、平台、客户、金额、支付状态、订单状态、日期
- 平台徽章显示来源（eBay AU / Amazon US 等）

### Inventory 库存管理
- 4 个统计卡片：总 SKU、低库存告警、缺货、仓库数
- 产品库存表格（待数据接入）

### Customer Service AI 客服
- 3 个统计卡片：总消息数、AI 处理数、需人工数
- 消息收件箱列表：发送者、平台、意图分类、AI 草稿
- 消息标签：需人工审核(橙色)

### Marketing 营销
- 营销活动卡片列表
- 6 种活动类型选择器
- "New Campaign" 按钮
- AI Generated 标签

### Analytics 数据分析
- 4 个 KPI 卡片：总收入、毛利、订单量、AI 成本
- 收入分平台图表（占位）
- 热销产品排名（占位）

### Settings 设置
- 平台连接卡片（eBay / Amazon / Shopify / TikTok）
- 每个平台显示支持区域、连接状态、Connect 按钮
- AI 配置：Anthropic / OpenAI API Key 状态

### Login 登录
- 邮箱 + 密码表单
- 登录 / 注册切换
- 错误提示
- 成功后跳转到 `/dashboard`

## API 客户端 (`lib/api.ts`)

### 功能

- Token 管理（localStorage 存储、自动附加 Authorization header）
- 401 自动跳转到 `/login`
- 统一错误处理
- 类型安全的 API 方法

### 可用方法

| 方法 | HTTP | 说明 |
|------|------|------|
| `api.login(email, password)` | POST /auth/login | 登录并存储 token |
| `api.register(email, password, name)` | POST /auth/register | 注册 |
| `api.getMe()` | GET /auth/me | 获取当前用户 |
| `api.listProducts(params)` | GET /products/ | 产品列表 |
| `api.getProduct(id)` | GET /products/{id} | 产品详情 |
| `api.createProduct(data)` | POST /products/ | 创建产品 |
| `api.updateProduct(id, data)` | PATCH /products/{id} | 更新产品 |
| `api.deleteProduct(id)` | DELETE /products/{id} | 删除产品 |
| `api.listOrders(params)` | GET /orders/ | 订单列表 |
| `api.getOrder(id)` | GET /orders/{id} | 订单详情 |
| `api.listListings(params)` | GET /listings/ | Listing 列表 |
| `api.getProductStock(productId)` | GET /inventory/products/{id} | 产品库存 |
| `api.adjustStock(data)` | POST /inventory/adjust | 调整库存 |
| `api.listLocations()` | GET /inventory/locations | 仓库列表 |
| `api.calculatePrice(data)` | POST /pricing/calculate | 计算建议价格 |
| `api.listCampaigns()` | GET /marketing/campaigns | 营销活动列表 |
| `api.createCampaign(data)` | POST /marketing/campaigns | 创建活动 |
| `api.listMessages(params)` | GET /customer-service/messages | 客服消息列表 |

### TypeScript 类型

文件底部定义了所有响应类型：`Product`, `Order`, `Listing`, `InventoryItem`, `Location`, `PriceCalculation`, `Campaign`, `Message`

## 前端环境变量

在 `frontend/.env.local` 中配置：

```
NEXT_PUBLIC_API_URL=http://localhost:8000/api/v1
```

生产环境：
```
NEXT_PUBLIC_API_URL=https://api.yourdomain.com/api/v1
```
