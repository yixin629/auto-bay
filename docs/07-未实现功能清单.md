# AutoBay 未实现功能清单

本文档列出原定计划中尚未完成的功能，按优先级和阶段分类。

---

## Phase 1 遗留（高优先级）

### 1.1 eBay OAuth 完整流程
- **现状**：EbayConnector 已实现 API 调用，但 OAuth2 授权码流程尚未在前端集成
- **需要做**：
  - 后端：`/api/v1/integrations/ebay/auth-url` 生成 eBay 授权 URL
  - 后端：`/api/v1/integrations/ebay/callback` 处理回调、交换 token
  - 前端：Settings 页面点击 "Connect" → 打开 eBay 授权页 → 回调存储 credentials
  - 自动 token 刷新（后台任务，token 过期前自动 refresh）
- **文件涉及**：新建 `backend/app/integrations/ebay/auth.py`、修改 `settings/page.tsx`

### 1.2 Amazon / Shopify / TikTok OAuth 流程
- 同上，每个平台需要独立的 OAuth 流程
- Amazon 需要额外的 AWS STS 角色签名
- TikTok 需要 shop_cipher 关联

### 1.3 Webhook 接收端点
- **现状**：各平台 Connector 支持轮询（poll），但缺少 Webhook 实时推送接收
- **需要做**：
  - `POST /api/v1/webhooks/ebay` — eBay Marketplace Account Deletion 和 Order Notification
  - `POST /api/v1/webhooks/shopify` — Shopify Webhook（order/create, product/update 等）
  - `POST /api/v1/webhooks/tiktok` — TikTok Shop Event Subscription
  - 每个 webhook 需要签名验证
- **文件涉及**：新建 `backend/app/integrations/{platform}/webhooks.py`

### 1.4 初始 Alembic 迁移执行
- **现状**：`alembic/env.py` 已配置、所有模型已注册，但 `versions/` 目录为空
- **需要做**：执行 `alembic revision --autogenerate -m "initial schema"` + `alembic upgrade head`
- **前置条件**：PostgreSQL 必须先启动

## Phase 2 功能（AI 上架 + 多平台）

### 2.1 AI Listing 一键生成端到端流程
- **现状**：AI agents 已实现（标题/描述/翻译），但缺少 API 端点和前端 UI 串联
- **需要做**：
  - 后端：`POST /api/v1/products/{id}/generate-listing` → 调用 listing_generator → 保存到 product_translations
  - 前端：产品详情页添加 "AI 生成" 按钮
  - 前端：显示 AI 生成的多语言标题/描述，支持编辑后确认
  - 前端：一键发布到所有已连接平台

### 2.2 产品详情页
- **现状**：只有产品列表页，缺少产品详情/编辑页面
- **需要做**：
  - `frontend/src/app/dashboard/products/[id]/page.tsx`
  - Tab 布局：概览（编辑）、Listings（各平台状态）、库存、AI（生成/翻译）
  - 产品图片上传和预览

### 2.3 跨平台库存同步的前端可视化
- **现状**：后台 Celery 任务已实现，但前端看不到同步状态
- **需要做**：
  - 库存页面显示每个产品在每个平台的实时库存
  - 同步状态指示器（最后同步时间、成功/失败）
  - 手动触发同步按钮

### 2.4 多语言产品翻译管理
- **现状**：`product_translations` 表和 `translate_listing()` agent 已存在
- **需要做**：
  - 前端 UI 管理翻译（查看/编辑 AI 翻译结果）
  - 支持的语言列表配置
  - 翻译质量标记（人工审核/未审核）

## Phase 3 功能（定价 + 供应链 + 物流）

### 3.1 定价控制台前端
- **现状**：后端定价 API 已实现（`/pricing/calculate`）
- **需要做**：
  - `frontend/src/app/dashboard/pricing/page.tsx`
  - 批量定价策略配置（每个 listing 设置策略）
  - 竞品价格输入和监控
  - 价格历史图表
  - 价格模拟计算器

### 3.2 竞品价格自动抓取
- **现状**：`competitor_match` 策略已实现，但竞品价格需手动输入
- **需要做**：
  - 通过 eBay Browse API 查询同类产品价格
  - 通过 Amazon Product Pricing API 获取竞品定价
  - 定期更新 listing.pricing_config.competitor_prices
  - Celery 任务：每日抓取竞品价格

### 3.3 1688 实际 API 对接
- **现状**：`Alibaba1688Adapter` 已实现接口，但使用 placeholder API URL
- **需要做**：
  - 对接实际的 1688 数据源（选项：DajiAPI / Xunyuantong / Oxylabs）
  - 或对接 AliExpress Affiliate API（有公开 API）
  - 产品搜索 → 利润评分 → 推荐列表
  - 从 1688 链接自动提取产品信息并创建 Product

### 3.4 物流发货自动化
- **现状**：`EasyPostAdapter` 已实现接口
- **需要做**：
  - 对接 EasyPost 实际 API（需要 API Key）
  - 订单详情页添加 "生成运单" 按钮
  - 自动选择最便宜的承运方式
  - 运单 PDF 生成和下载
  - 海关申报信息自动填充（基于产品的 hs_code）

### 3.5 4PX / YunExpress 跨境物流集成
- **现状**：未实现
- **需要做**：
  - `backend/app/modules/logistics/carriers/fourpx.py`
  - `backend/app/modules/logistics/carriers/yunexpress.py`
  - 中国仓库直发到全球的物流方案
  - 支持 Dropship 模式下供应商直发

### 3.6 订单自动处理工作流
- **现状**：订单同步已实现，但后续自动化缺失
- **需要做**：
  - 新订单 → 自动预留库存 → 自动生成运单 → 自动上传物流号 → 自动标记发货
  - Dropship 模式：新订单 → 自动下单给供应商
  - 低库存自动提醒/自动补货

## Phase 4 功能（客服 + 营销）

### 4.1 客服消息实际收发
- **现状**：AI 分类和回复生成已实现，但消息没有实际从平台收发
- **需要做**：
  - eBay: 集成 Post-Order API / Member Message API
  - Amazon: 集成 Messaging API
  - Shopify: 通过 Webhook 接收（无原生 messaging API）
  - TikTok: 集成 Conversation API
  - Celery 任务：定期拉取各平台新消息

### 4.2 客服消息实时推送
- **现状**：前端轮询查看消息
- **需要做**：
  - WebSocket 连接（`/ws/customer-service`）
  - 新消息即时推送到前端
  - 浏览器通知

### 4.3 退换货自动化
- **现状**：OrderStatus 有 return_requested / returned / refunded 状态
- **需要做**：
  - 客服 AI 检测到退货请求 → 自动生成退货标签
  - 退款审批流程（自动/手动）
  - 退货原因统计和分析

### 4.4 营销活动实际投放
- **现状**：Campaign 模型和 AI 文案生成已实现
- **需要做**：
  - eBay Promoted Listings API 对接 → 自动创建推广
  - Amazon PPC API 对接 → 自动创建 Sponsored Products
  - Google Ads API 对接 → 自动创建搜索广告
  - Meta Ads API 对接 → 自动创建 Facebook/Instagram 广告
  - TikTok Ads API 对接
  - 广告效果数据回收（ROAS、CTR、CPA）
  - 预算自动优化（基于 ROAS 分配）

### 4.5 SEO 自动优化
- **现状**：`generate_seo_keywords()` 已实现
- **需要做**：
  - 定期检查 Listing 标题/描述是否包含高价值关键词
  - 自动建议优化（不自动修改，人工确认）
  - 竞品 SEO 分析

### 4.6 社交媒体自动发布
- **现状**：`generate_social_post()` 已实现
- **需要做**：
  - Instagram/TikTok/Twitter API 对接
  - 自动生成产品推广视频脚本
  - 内容日历管理
