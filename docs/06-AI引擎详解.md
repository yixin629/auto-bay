## 2. LLM 网关 (`app/ai/llm.py`)

### 模型映射

| Tier | 模型 | 输入成本 ($/1K tokens) | 输出成本 ($/1K tokens) | 用途 |
|------|------|----------------------|----------------------|------|
| BUDGET | claude-haiku-4-5 | $0.001 | $0.005 | 分类、标题、关键词 |
| STANDARD | claude-sonnet-4-6 | $0.003 | $0.015 | 描述、回复、广告 |
| PREMIUM | claude-opus-4-6 | $0.015 | $0.075 | 复杂分析、趋势综合 |

### 调用示例

```python
from app.ai.llm import llm_client, ModelTier

result = await llm_client.complete(
    messages=[{"role": "user", "content": "Generate a title for..."}],
    tier=ModelTier.BUDGET,
    system="You are an eBay title optimizer...",
    max_tokens=100,
    temperature=0.3,
)

print(result.content)        # 生成的文本
print(result.model)          # claude-haiku-4-5-20251001
print(result.input_tokens)   # 输入 token 数
print(result.output_tokens)  # 输出 token 数
print(result.cost_usd)       # 本次调用成本（美元）
print(result.latency_ms)     # 延迟（毫秒）
```

### 成本追踪

每次 LLM 调用自动记录到 `ai_tasks` 表：model_used、tokens_used、cost_usd。可在 Analytics 页面查看月度 AI 支出。

## 3. AI Agents 详解

### 3.1 Listing Generator (`agents/listing_generator.py`)

**功能**：为不同平台生成 SEO 优化的标题和描述

| 函数 | Tier | 用途 | max_tokens |
|------|------|------|-----------|
| `generate_listing_title()` | BUDGET | 平台特定标题（eBay 80字符 / Amazon 200字符） | 100 |
| `generate_listing_description()` | STANDARD | HTML 格式产品描述，150-300 词 | 800 |
| `translate_listing()` | STANDARD | 标题+描述翻译（保持 SEO 质量） | 1000 |

**标题生成 Prompt 策略**：
- 前置高搜索量关键词
- 包含品牌、型号、关键规格
- 遵守平台字符限制
- 不用全大写或过度标点

**描述生成 Prompt 策略**：
- 要点列表突出特性和优势
- 自然融入 SEO 关键词
- 解答常见买家疑虑
- 输出 HTML 格式

### 3.2 Customer Support (`agents/customer_support.py`)

**功能**：自动分类客户消息意图 + 生成回复

**流程**：
```
收到客户消息
    │
    ▼
classify_intent() ← BUDGET 模型，~50ms
    │
    ├── shipping_status  → 查订单状态 → 生成物流回复
    ├── return_request   → 引导退货流程
    ├── product_question → 查产品信息 → 生成答复
    ├── complaint        → 道歉 + 解决方案
    └── general          → 通用回复
    │
    ▼
generate_response() ← STANDARD 模型
    │
    ▼
confidence >= 0.85 ?
    ├── YES → 自动发送
    └── NO  → 标记 requires_human，等待人工审批
```

**意图分类输出格式**：
```json
{"intent": "shipping_status", "confidence": 0.92}
```

**回复输出格式**：
```json
{
  "response": "Hi! Your order #12345 was shipped on...",
  "suggested_actions": ["send_tracking_link", "follow_up_3_days"]
}
```

### 3.3 Product Research (`agents/product_research.py`)

**功能**：产品机会评估

**利润评分（规则引擎，不用 LLM）**：
```
score = margin_weight × 0.30
      + demand_signal × 0.25
      + (1 - competition) × 0.20
      + trend_velocity × 0.15
      + supplier_reliability × 0.10
```

**机会分析（PREMIUM 模型）**：
- 只对 Top 20 高分产品调用
- 输出：可行性评估、风险点、建议策略、推荐平台

### 3.4 Pricing Optimizer (`agents/pricing_optimizer.py`)

**功能**：价格异常检测 + 定价解释

- `detect_pricing_anomalies()` — **纯规则**，检测低毛利、高毛利、偏离市场均价
- `explain_pricing_recommendation()` — BUDGET 模型，生成人可读的定价理由

### 3.5 Marketing Agent (`agents/marketing_agent.py`)

**功能**：广告文案、SEO 关键词、社媒内容

| 函数 | Tier | 输出 |
|------|------|------|
| `generate_ad_copy()` | STANDARD | headlines[], descriptions[], call_to_action |
| `generate_seo_keywords()` | BUDGET | primary_keywords[], long_tail_keywords[], negative_keywords[] |
| `generate_social_post()` | BUDGET | caption, hashtags[], call_to_action |

**平台适配**：
- Google Ads: headline 30字符 / description 90字符
- Facebook: primary text 125字符
- TikTok: 短平快 + emoji
- eBay Promoted: 强调价值和包邮
- Amazon PPC: 关键词密集

## 4. LLM vs 规则引擎决策表

| 功能 | 方法 | 理由 |
|------|------|------|
| 产品标题生成 | LLM (BUDGET) | 需要自然语言创意 |
| 产品描述生成 | LLM (STANDARD) | 需要创意 + SEO + HTML |
| 翻译 | LLM (STANDARD) | 需要保持营销语气 |
| 意图分类 | LLM (BUDGET) | 自然语言理解 |
| 客服回复 | LLM (STANDARD) | 需要同理心 + 准确 |
| 趋势分析综合 | LLM (PREMIUM) | 多因素复杂推理 |
| 广告文案 | LLM (STANDARD) | 创意 + 平台规则 |
| SEO 关键词 | LLM (BUDGET) | 语义理解 |
| **利润评分** | **规则引擎** | 加权公式，确定性计算 |
| **动态定价** | **规则引擎** | cost_plus / competitor_match |
| **汇率转换** | **API + 数学** | exchangerate.host API |
| **库存同步** | **CRUD** | 纯数据库操作 |
| **预算优化** | **ROAS 公式** | 线性优化 |
| **价格异常检测** | **规则引擎** | 阈值比较 |
| **分类映射** | **查找表** | 80% 直接匹配，20% fallback LLM |

## 5. 成本预估（月度，1000 产品规模）

| 任务 | 月调用量 | 模型 | 估算成本 |
|------|---------|------|---------|
| 产品标题生成 | 1,000 | BUDGET | ~$0.50 |
| 产品描述 (EN+CN) | 2,000 | STANDARD | ~$6.00 |
| 趋势分析综合 | 100 | PREMIUM | ~$5.00 |
| 客服意图分类 | 10,000 | BUDGET | ~$2.00 |
| 客服回复生成 | 5,000 | STANDARD | ~$15.00 |
| 广告文案 | 500 | STANDARD | ~$2.00 |
| 社媒内容 | 1,000 | BUDGET | ~$1.00 |
| **月度总计** | | | **~$31.50** |

### 成本优化策略

1. **Prompt 缓存**：Anthropic 对重复的 system prompt 收 10% 费用（vs 100%）
2. **批量 API**：非实时任务用 Batch API，成本减半
3. **严格 max_tokens**：标题限 100 tokens，不用默认 4096
4. **短路模式**：客服如果检测到物流号查询（>95% 信心度），直接返回数据不调 LLM
5. **月度预算上限**：可配置 soft/hard limit，超过后降级或暂停
